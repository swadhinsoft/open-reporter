Features

Everything in a single HTML file

Reports are fully self-contained — no server, no external assets needed. Share a single .html file and it works everywhere.

Interactive Dashboard

Doughnut chart for test distribution, pass-rate ring, and per-browser bar chart — all rendered client-side with Chart.js.

Auto Screenshots

On test failure the registered WebDriver / AppiumDriver captures a screenshot, embeds it as base64, and shows a zoomable thumbnail in the expanded row.

Dark & Light Mode

GitHub-style dark theme by default. One-click toggle persisted in localStorage — no flash on reload.

Browser & OS Icons

Chrome, Firefox, Edge, Safari, Opera brand icons. Windows, macOS, Linux, Android OS icons via Font Awesome — inferred from browser when OS is unknown.

Search & Filter

Live search across test name, description, browser, and OS. One-click filters for Passed / Failed / Skipped with animated counts.

Excel & PDF Export

Export the full result table to .xlsx via SheetJS or a formatted .pdf via jsPDF + AutoTable — with coloured status cells.

Custom Project Logo

Point logo in openreporter.json to a PNG/JPG/SVG — it's embedded as base64 in the report header. Supports relative and absolute paths.

Framework-Agnostic Core

The core (OpenReporter singleton) has zero framework imports. Adapters for TestNG, JUnit 5, and Cucumber plug into each framework's event system independently.

Thread-Safe Parallel Execution

ThreadLocal is used for driver, browser, and current-test state — safe for any number of parallel threads in TestNG / JUnit / Cucumber.

Installation

Two steps, then you're done

OpenReporter is distributed via JitPack. No account or token required — just add the repository and the dependency.

1
Add JitPack repository to pom.xml
<repositories> <repository> <id>jitpack.io</id> <url>https://jitpack.io</url> </repository> </repositories>
Maven
2
Add the dependency
<dependency> <groupId>com.github.swadhinsoft</groupId> <artifactId>open-reporter</artifactId> <version>1.0.0</version> <scope>test</scope> </dependency>
Maven · scope: test
Gradle (Kotlin DSL):
testImplementation("com.github.swadhinsoft:open-reporter:1.0.0") — add maven { url = uri("https://jitpack.io") } to your repositories block first.
Integration

Pick your framework

Each adapter hooks into its framework's native event system. You don't need to change your test logic — just wire the adapter once.

1
Add listener to testng.xml
<suite name="MySuite"> <listeners> <listener class-name="io.github.swadhinsoft.openreporter.adapter.TestNGListener"/> </listeners> ... </suite>
2
Register driver in your base class
import io.github.swadhinsoft.openreporter.OpenReporter; public class BaseTest { private static final ThreadLocal<WebDriver> driverTL = new ThreadLocal<>(); // helper used by tests public WebDriver getDriver() { return driverTL.get(); } @BeforeMethod public void setUp(@Optional("chrome") String browser) { WebDriver driver = createDriver(browser); driverTL.set(driver); OpenReporter.getInstance().registerDriver(driver); OpenReporter.getInstance().setBrowser(browser); } @AfterMethod public void tearDown() { OpenReporter.getInstance().unregisterDriver(); WebDriver d = driverTL.get(); if (d != null) { d.quit(); driverTL.remove(); } } }
Parallel execution: use parallel="tests" or parallel="methods" in testng.xml — OpenReporter is ThreadLocal-safe.
1
Annotate your test class
import io.github.swadhinsoft.openreporter.adapter.JUnit5Extension; import org.junit.jupiter.api.extension.ExtendWith; @ExtendWith(JUnit5Extension.class) public class SearchTest { // your tests here }
2
Register driver in @BeforeEach / @AfterEach
@BeforeEach void setUp() { driver = new ChromeDriver(); OpenReporter.getInstance().registerDriver(driver); OpenReporter.getInstance().setBrowser("chrome"); } @AfterEach void tearDown() { OpenReporter.getInstance().unregisterDriver(); if (driver != null) driver.quit(); }
Report flush: JUnit5Extension calls generateReport() in @AfterAll — one report per test class. For a combined report across multiple classes, call OpenReporter.getInstance().generateReport() from a TestExecutionListener after the full run.
1
Add plugin to @CucumberOptions
@CucumberOptions( features = "src/test/resources/features", glue = "com.example.steps", plugin = { "pretty", "io.github.swadhinsoft.openreporter.adapter.CucumberPlugin" } ) public class TestRunner {}
2
Register driver in Cucumber @Before / @After
@Before public void beforeScenario() { driver = new ChromeDriver(); OpenReporter.getInstance().registerDriver(driver); OpenReporter.getInstance().setBrowser("chrome"); } @After public void afterScenario() { OpenReporter.getInstance().unregisterDriver(); if (driver != null) driver.quit(); }
Report path: target/open-reporter/report.html by default. Change it via outputDir in openreporter.json.
TestNG
TestNGListener
Implements ITestListener + ISuiteListener. Report generated on suite finish.
JUnit 5
JUnit5Extension
Implements BeforeEachCallback, TestWatcher, AfterAllCallback.
Cucumber 7
CucumberPlugin
Implements io.cucumber.plugin.EventListener. Report on TestRunFinished.
Selenium / Appium
registerDriver(driver)
Any class implementing WebDriver — Selenium and Appium both work.
Configuration

openreporter.json

Drop openreporter.json in your project root. All fields are optional — sensible defaults apply.

openreporter.json (project root)
{ "title": "My Project Test Report", "logo": "src/test/resources/logo.png", "outputDir": "target/open-reporter", "environment": "QA" }
Relative paths resolve from project root
Override path via JVM property
# Maven Surefire <configuration> <systemPropertyVariables> <openreporter.config> /ci/config/openreporter.json </openreporter.config> </systemPropertyVariables> </configuration> # Or on the command line mvn test -Dopenreporter.config=/path/to/file.json
KeyDescriptionDefault
titleReport page title shown in browser tab and header"Test Execution Report"
logoPath to project logo (PNG / JPG / SVG / GIF). Embedded as base64, displayed at 60×60 px in the header"" (uses default icon)
outputDirDirectory where report.html is written"target/open-reporter"
environmentEnvironment label shown in the report (e.g. QA, Staging, Prod)"—"
Lookup order: -Dopenreporter.config JVM property → openreporter.json in user.dir (project root) → openreporter.json on classpath → built-in defaults. The reporter never fails if no file is found.
Report Features

What's inside the report

Every report is a standalone HTML file you can open in any browser, attach to a CI artifact, or email directly.

Summary & Charts

  • 5 summary cards: Total / Passed / Failed / Skipped / Duration
  • Animated doughnut chart (Chart.js)
  • Pass-rate ring with conic-gradient
  • Per-browser test count bars

Test Table

  • Expandable rows: logs, error, stack trace, screenshot
  • Browser brand icon + OS icon per row
  • Live search (name / description / browser / OS)
  • Passed / Failed / Skipped filter buttons

Screenshots

  • Captured automatically on failure
  • Embedded as base64 — no external files needed
  • Thumbnail in expanded row, click to zoom (lightbox)
  • Works with any WebDriver: browser or Appium mobile

Export

  • Excel (.xlsx) — SheetJS, column widths, header styling
  • PDF — jsPDF + AutoTable, colour-coded status cells
  • Dark / Light mode toggle (persisted in localStorage)
  • Custom logo embedded from local file path
CI / CD

GitHub Actions — upload report as artifact

Reports land in target/open-reporter/ by default. Attach them as GitHub Actions artifacts so every run has a downloadable report.

.github/workflows/tests.yml (excerpt)
- name: Run tests run: mvn test - name: Upload OpenReporter HTML report uses: actions/upload-artifact@v4 if: always() # upload even on failure with: name: open-reporter-${{ github.run_number }} path: target/open-reporter/ retention-days: 14
Place after your test step
